# Token Notifier 설치 안내

macOS 메뉴바에서 Claude Code / Codex 사용량을 보여주는 앱입니다.

## 0. 먼저 확인

이 앱은 **새로 로그인을 받지 않고**, 이미 설치된 CLI 의 로그인 정보를 그대로 읽어옵니다.
아래 중 최소 하나가 로그인돼 있어야 숫자가 보입니다.

- Claude Code CLI 로그인 (`claude` 사용 중이면 보통 돼 있음)
- Codex(ChatGPT) CLI 로그인 (`~/.codex/auth.json` 존재)

둘 다 없으면 메뉴바 아이콘은 떠도 사용량이 빈 채로 표시됩니다.

## 1. 설치

1. 받은 `Token-Notifier-x.x.x.zip` 의 압축을 풉니다.
2. 나온 `Token Notifier.app` 을 `응용 프로그램(Applications)` 폴더로 옮깁니다.

## 2. 첫 실행 (Gatekeeper 우회 - 한 번만)

이 앱은 Apple 공증을 받지 않아 처음엔 macOS 가 실행을 막습니다. 둘 중 하나로 엽니다.

### 방법 A - 터미널 한 줄 (가장 확실)

`터미널` 앱을 열고 아래를 붙여넣어 실행합니다.

```bash
xattr -dr com.apple.quarantine "/Applications/Token Notifier.app"
```

그 다음 앱을 더블클릭하면 정상 실행됩니다.

### 방법 B - 시스템 설정에서 허용

1. `Token Notifier.app` 을 그냥 더블클릭합니다. (차단 경고가 뜸 - 닫기)
2. `시스템 설정 → 개인정보 보호 및 보안` 으로 이동합니다.
3. 아래쪽 보안 항목에 "Token Notifier 가 차단되었습니다" 옆의 **"확인 없이 열기"** 를 누릅니다.
4. 다시 앱을 열고 경고창에서 **"열기"** 를 누릅니다.

## 3. 권한

- 첫 실행 시 **알림(Notification)** 권한을 물으면 허용하면 사용량 임계치 알림을 받습니다.
- 로그인 시 자동 실행을 원하면 앱 메뉴/시스템 설정의 로그인 항목에서 켤 수 있습니다.

## 문제가 있을 때

- 메뉴바 아이콘은 보이는데 숫자가 비어 있음 → 위 "0. 먼저 확인" 의 CLI 로그인 상태를 점검하세요.
- "손상되었기 때문에 열 수 없습니다" → 방법 A 의 `xattr` 명령을 실행하세요.
